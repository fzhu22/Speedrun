## DOK 3 - Insights (revised, validated)

**Insight 1 (TESTABLE-HYPOTHESIS / MEASUREMENT-strong): The highest-leverage bet is not AI card generation; it is a student-driven miss-review + authoring scaffold that makes explanations hard to vary.** The tool perturbs the surface, prompts the underlying principle, flags traps, and asks for a CONJECTURE-FIRST card whose load-bearing field is a DISCONFIRMER ("what one fact would flip this answer?"); the STUDENT still produces the card (automating the body forfeits the generation advantage AND collapses into the faulty AI card-gen the corpus rejects). Strongest version adds a boundary case against the student's stated principle (record what the explanation RULES OUT, not just recalls). *Rests on:* Pan 2023 (d=0.45/0.29); Bertsch 2007 (~.40); Rowland 2014; BMC "all faulty"; parent transfer moderators + parent SPOV 11 WEAK; cogsci hard-to-vary/disconfirmer. *Changed:* "card-construction compiler"→student-authored scaffold; "killer feature"→highest-leverage bet in a contested regime.

**Insight 2 (BUILDABILITY-CONSTRAINT / TESTABLE-HYPOTHESIS): "More reviews" is a false lever only for APPLICATION/transfer, not for memory.** Volume/spacing/adaptive scheduling DO move surface memory (don't pretend FSRS is broken); the feature-layer delta is narrower — exam-horizon desired-retention ramps, application queues, failure-triggered remediation layered over FSRS. True queue RE-RANKING for concept/application priority is fork-only; Anki custom scheduling re-weights a per-card next interval, it cannot reorder the queue from semantic context. *Rests on:* Donoghue & Hattie d=0.85/0.74; Eglington & Pavlik +40%; Lindsey +16.5%/+10%; FSRS Helper/FSRS-6/per-deck DR; custom-scheduler context `{deck_name, seed}` + per-card re-weight limit; parent Insight 7. *Changed:* scoped to transfer; preserved spacing's memory benefit; moved upstream-card-construction into Insight 1.

**Insight 3 (TESTABLE-HYPOTHESIS / MEASUREMENT-strong): Application cards are only a testable transfer hypothesis if they preserve deep structure while perturbing surface cues AND THEN verify representation shift.** Near discrimination among confusable science concepts is defensible; passage-level MCAT reasoning transfer is contested. Pair surface-varied minivignettes with a DEEP-STRUCTURE SORT probe ("which two share a principle?" / "which doesn't belong?"); perturbed-item accuracy alone is insufficient (students learn incidental shortcuts). *Rests on:* parent transfer d=0.40 + moderators / far ~0; parent SPOV 11 WEAK (words −0.39, expository ns); Agarwal transfer-appropriate processing; Koriat/Bjork cue-dependency; Williams/Lombrozo shortcut risk [critique-sourced]. *Changed:* marked hypothesis; separated near-discrimination from passage transfer; added deep-structure probe.

**Insight 4 (TESTABLE-HYPOTHESIS / BUILDABILITY-CONSTRAINT — strongest feature-novel): The strongest novel feature is CURATED discrimination drills for known MCAT confusions, not generic interleaving.** Use human-reviewed confusable sets / a curated concept graph (NOT auto-tagging, which re-imports faulty-AI risk), generated after baseline competence; each drill forces naming the discriminating MECHANISM + a falsifiable consequence (not just the category label). Mechanism plausibly transfers to confusable science clusters; magnitude untested for MCAT (don't import Rohrer math d=0.83 as the expected effect). *Rests on:* Brunmair & Richter (math +, words −0.39, expository ns); Rohrer/Taylor as mechanism not MCAT magnitude; learner blocking preference; Anki lacks semantic-topic interleaving; engineer: review-interleaving is fork-only (build as desktop-prepare drill gen + cross-platform review). *Changed:* removed auto-tagging + inflated transfer; specified curated sets + mechanism/consequence answers.

**Insight 5 (BUILDABILITY-CONSTRAINT, inherited): CARS forces the product to decide whether it is a flashcard fork or a broader MCAT practice system.** Recall offers NO diagnostic signal for the reasoning/timing/tone/argument failures that limit CARS; the real feature decision is build a separate passage/reasoning module (with error-pattern tracking) vs explicitly leave CARS to integrations (Jack Westin) and keep the fork science-content-focused. *Rests on:* parent CARS no-content + section-aware; practitioner consensus; Pan far-transfer boundaries. *Changed:* marked inherited; softened "false confidence"→"no diagnostic signal for the limiting failure."

**Insight 6 (TESTABLE-HYPOTHESIS / MEASUREMENT-strong): Difficulty should FADE SUPPORT inside validated concept families, not globally escalate to "hard mode."** For problem-solving/high-element-interactivity content, fade only after ≥2 unaided successes + 1 transfer item within a family; fade TOWARD explaining the rejected alternatives / counterfactual branches (expose the causal skeleton early). Needs the same per-concept expertise estimator as Insight 4; weak fit for declarative recall (amino-acid properties / psych terms). *Rests on:* Tetzlaff 2025 (d=0.505 / −0.428); Sweller/Chen/Kalyuga; Atkinson/Renkl fading+principle prompts; Karpicke 2025; cogsci counterfactual/rules-out. *Changed:* operationalized the fade threshold; scoped to high-element content; tied to per-concept expertise estimation.

**Insight 7 (MEASUREMENT-strong / TESTABLE-HYPOTHESIS): Multiple-choice practice can be net-NEGATIVE without distractor-level feedback + remediation.** Not "every card needs an explanation" (unscaffolded self-explanation is null/harmful); the distinctive feature = feedback that forces ERROR-CLASSIFICATION + rewrite/remediation with SOURCE-BACKED distractor rationales (why the lure was tempting, what model it exposes, what replacement drill follows) — while managing load (over-atomized remediation = review debt). *Rests on:* Butler & Roediger lures; Butler/Karpicke/Roediger feedback; Rowland g=0.03 ≤50%; McCarthy/iSTART null/detrimental; Kluger & DeNisi (~⅓ hurt) [critique-sourced]; med-ed vignette critiques. *Changed:* strengthened to net-negative; removed universal-explanation rule; merged confidence-before-reveal into Insight 8.

**Insight 8 (MEASUREMENT-strong — cleanest pass): Calibration is valuable only when it captures committed-answer mismatches and routes high-confidence-wrong to misconception remediation.** Score confidence ONLY after a committed answer (don't reward hedging — Goodhart guard); record WHICH false model drove the high-confidence error; the jewel = high-confidence-wrong → source-backed misconception drill; never use confidence alone to change FSRS intervals. The mismatch-routing taxonomy is an UNTESTED operational hypothesis. *Rests on:* Nelson & Narens; Koriat JOL biases; Lee 2025 +8.9%/+4.1% [thin — not load-bearing alone]; Butler feedback; parent score-honesty separation; cogsci Goodhart guard. *Changed:* labeled routing untested; added committed-answer guard + false-model capture.

**Insight 9 (TESTABLE-HYPOTHESIS): The problem with gamification is MECHANISM DISTORTION, not "gamification is bad."** Net-positive gamification metas mean don't reject the class; the narrow claim: engagement/competition/streak-optimizing mechanics can PUNISH the behaviors MCAT gains require (confronting errors, hard reviews, admitting low confidence, remediation time) — and net-positive metas don't rebut it (they measure engagement/short-term/surface, not error-confrontation/transfer). Use NON-DISTORTING gamification: private mastery/coverage, recovery streaks, effort-credit for hard items, verbal encouragement; NO public ranking / streak-loss penalty. *Rests on:* Sailer & Homner 2020 g≈0.49 / Huang 2020 g≈.464 / Frontiers 2023 g≈.82 [critique-sourced]; Deci 1999 (tangible reward caveat, NOT all gamification); Almeida 2023; Huang 2024 LBL-negative; streak-anxiety. *Changed:* fixed Deci category slip; added net-positive metas; narrowed to mechanism-level Goodhart claim.

**Insight 10 (TESTABLE-HYPOTHESIS, inherited): AI should GENERATE severe tests against a human-authored key, not CERTIFY understanding.** AI can produce counterexamples/boundary cases/contradiction checks/disconfirmer prompts, but must NOT grade explanation quality as authoritative (resemblance-to-key is Goodhartable). Test the AI lane against the CRUTCH EFFECT: if assisted performance rises while UNASSISTED MCAT-style performance falls, the feature fails even if students like it (the MCAT is taken unassisted). Pro-AI evidence is thin/preprint. *Rests on:* Chen 2026 +11.9pp (MC ns) [preprint]; LearnLM/Eedi +5.5pp / 76.4% [preprint]; BMC "all faulty"; GPT-4 +48% assisted / −17% unassisted [preprint, critique-sourced]; SmartFlash editability; parent AI-lane separation. *Changed:* cut AI grading authority; restricted to checkable criticism vs a human key; made unassisted degradation the load-bearing risk.

**Insight 11 (MEASUREMENT-strong — NEW, metric layer): The fork should make its OWN theory of learning falsifiable with discrimination + Preparation-for-Future-Learning metrics.** Perturbed-item accuracy is too weak a KPI (gameable by surface shortcuts; capped at sequestered problem-solving); better transfer KPIs = empirical item DISCRIMINATION, boundary/competing-principle item performance, and a PFL proxy ("after remediation, fewer trials to acquire the NEXT concept in the cluster"). This is what stops "learning-science features" from being unfalsifiable UX decoration. *Rests on:* Bransford & Schwartz PFL [critique-sourced]; Williams/Lombrozo shortcut/illusory patterns [critique-sourced]; parent transfer boundaries; Insight 3 deep-structure probe; Insight 4 clusters. *Changed:* NEW (cogsci) — converts transfer claims into measurable product-level falsifiers.

**Insight 12 (BUILDABILITY-CONSTRAINT — NEW, architecture): Buildability forces a TWO-TIER architecture — desktop "prepare," cross-platform "review."** The desktop prepare tier (Python/Rust, sees the whole collection) emits small syncable artifacts (tags / a companion store); the review tier is card-template JavaScript that works on AnkiMobile + AnkiDroid. Mandatory because: `customData` <100 bytes / 8-byte keys; custom scheduling sees only `{deck_name, seed}` + re-weights a per-card next interval (NOT a queue re-rank); AnkiMobile no add-ons / AnkiDroid no Python. True concept-interleaving + semantic queue re-ranking are FORK-ONLY; front-side incremental cloze is a TEMPLATE problem (Anki-Cloze-Interactive), not a scheduler feature. *Rests on:* engineer code-verified Anki constraints. *Changed:* NEW — determines which feature claims can be cross-platform vs desktop-prepared vs fork-only.
