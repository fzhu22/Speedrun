## Purpose

**Purpose (North Star).** Determine the spiky, defensible, *feature-level* positions a team should adopt when forking Anki for the MCAT: **which learning-science features to build to push studying past recognition-level recall toward exam-style application, reasoning, and transfer; how to implement each on Anki's real substrate** (Rust `rslib` / `fsrs-rs`, global custom-scheduling JavaScript, `customData`, card templates, cross-platform AnkiMobile/AnkiDroid constraints); **and which proposed features backfire or are cargo-cult.** The deliverable is reusable build context: each position names the concrete feature, where it physically runs, why it is non-redundant against the existing stack (AnKing + UWorld + Jack Westin), and the dated experiment or buildability check that would settle it.

### In Scope
- **Card-design & retrieval features:** student-authored miss→card scaffolds, disconfirmer fields, generation vs auto-gen, application vs recognition cards.
- **Transfer engineering:** deep-structure sort gates, curated mechanism-discrimination drills, surface perturbation, the CARS boundary.
- **Difficulty & feedback features:** per-family support-fading vs "hard mode," MC distractor-feedback + error-classification engines.
- **Calibration & motivation features:** committed-answer confidence capture + high-confidence-wrong routing; non-distorting vs distorting gamification.
- **AI's allowed lane:** generating severe tests vs grading; the assisted-up/unassisted-down crutch kill-switch.
- **Scheduling-policy features layered over FSRS:** exam-date desired-retention ramps.
- **The metric & architecture layer:** falsifiable transfer KPIs (item discrimination + PFL); the two-tier desktop-prepare → cross-platform-review architecture.

### Out of Scope
- **The three-score honesty / readiness architecture** (Memory / Performance / Readiness, ranges, the give-up rule, leakage-controlled evaluation) — that is the **sibling run** (`mcat-anki-study-app`), reused here as input, not re-debated.
- **The fixed project rules themselves** (fork Anki; shared Rust engine; AGPL; MCAT-specific) — givens, not debated.
- **Non-MCAT exams** (USMLE, bar, NCLEX, LSAT/GRE appear only as out-of-distribution *reach* checks, never as targets).
- **Final production code, UI visual design, and pricing** — this is reusable decision context, not an implementation.

### VALIDATED — Test 1 spiky · Test 2 crux + retreat-resistance + reach pass · Test 3 HOLDS-ESTABLISHED (lead the menu)

**Spiky POV 4 (Validated) — Every concept-aware feature must be two-tier (desktop "prepare" → cross-platform "review"), because live on-device semantic queue re-ranking through stock Anki is code-verified impossible.** A desktop "prepare" tier (Python/Rust, sees the whole collection) emits small syncable artifacts; the "review" tier is card-template JavaScript that runs on AnkiMobile + AnkiDroid. Any "your phone reorders your study queue by concept in real time" promise is **vaporware** — stock Anki/AnkiMobile custom-scheduling JS can re-weight a single card's next interval, but it cannot reorder the due queue by semantic concept cluster.

Elaboration: This is the architecture spine under SPOVs 2, 3, 10 and 11 — it dictates *where every smart-sequencing feature physically runs*. The substrate is fixed and code-verified: custom scheduling is **global JavaScript** that sees only `SchedulingContext{deck_name, seed}`, runs at *answer-time* on the current card's `states`, and cannot reach the queue the backend already built; `customData` is **≤100 bytes / 8-byte keys**; the v3 scheduler **cannot be monkey-patched**; AnkiMobile allows **no add-ons** and AnkiDroid allows **no Python** (JS only); per-card desired retention is "unaccessible to custom scheduling… only via an add-on" (Anki forum). Therefore true concept-interleaving and semantic re-ranking are **fork-only and must be desktop-prepared**, while front-side incremental cloze is a *template* problem (Anki-Cloze-Interactive / Closet), not a scheduler feature. This is **non-redundant by construction** — AnKing (content), UWorld (QBank), and Jack Westin (CARS) are content/practice layers and say nothing about the fork's engine reality; it is a buildability truth a fork must internalize before it over-promises. Expert tension: aligns with Damien Elmes + Jarrett Ye/Expertium on what the engine actually exposes; disagrees with any marketing implying live on-device semantic re-ranking.

Prediction (or Disconfirmer): **Buildability, checkable now (re-verify by 2026-12-31):** no working build exists in which live AnkiMobile custom-scheduling JS **REORDERS** the due queue by semantic concept cluster (not merely re-weights one card's next interval) using only `{deck_name, seed}` + `customData` ≤100 bytes with **no** desktop-prepared artifact. Non-retreatable: re-weighting a single interval, or a front-side cloze via template JS, does **NOT** count as a queue re-rank. If such a build exists, "two-tier is mandatory" is false.

How to resolve it: **Already resolved by primary source — re-runnable today, no experiment needed.** Read Anki's `scheduler.proto` (`SchedulingContext` exposes only `{deck_name, seed}`), `rslib/data.rs` (`customData` ≤100 bytes / 8-byte keys), and the v3-scheduler FAQ (the due queue is built backend-side; custom JS runs only at answer-time on the current card). Confirm per-card DR is reachable only via an add-on (Anki forum). The crux is settled by the engine itself.

Validation: Test 1 — cold models disputed (GPT 5/dispute, Claude dispute) → PASS. Test 2 — crux PASS, retreat-resistance PASS (scoped to *stock Anki/AnkiMobile custom-scheduling JS alone*, not "any SRS fork"), reach PASS (any cross-platform SRS fork wanting on-device semantic re-ranking). Test 3 — **HOLDS-ESTABLISHED** against Anki source (`scheduler.proto` + `rslib/data.rs` + v3 FAQ + forum). This is the jackpot: a true-but-not-widely-known buildability fact, established by primary source, that the rest of the menu builds on.

### STRONG — Test 1 spiky · Test 2 crux + retreat-resistance + reach pass · Test 3 HOLDS-PENDING

**Spiky POV 2 (Strong) — An "application card" earns the name only if the student passes a held-out deep-structure SORT; ship the sort as the GATE that unlocks "transfer-trained" status.** Perturbed-item accuracy alone certifies a *pattern-match*, not transfer. Pair surface-varied minivignettes with a deep-structure sort probe ("which two share the principle?" / "which doesn't belong?") and only mark a concept "transfer-trained" once the learner passes the sort on held-out items.

Elaboration: The concrete feature is a **sort gate**: after a student drills surface-varied application cards on a concept, the fork serves a *held-out* sort task whose items the student never studied; passing it (not just high perturbed accuracy) flips the concept's status flag. On Anki's substrate this is a **desktop-prepared** artifact (the sort sets and the held-out split are generated where the whole collection is visible) surfaced through review-tier template JS — exactly the two-tier pattern SPOV 4 makes mandatory. It is **non-redundant**: UWorld gives post-hoc rationales, AnKing gives cloze recall, Jack Westin gives passages, but none **gate** a "you have actually trained the principle" status on a held-out principle-sort. Evidence: Chi's expert/novice deep-vs-surface categorization + transfer-appropriate processing say transfer is a *representation shift*, and Pan & Rickard show recall→transfer is conditional (d=0.40; ~0 without moderators) — so accuracy on studied perturbations can be an incidental shortcut. Expert tension: extends Chi + Pan; disagrees with heatmap/streak culture and AI-minivignette vendors that treat more questions as more transfer.

Prediction (or Disconfirmer): By **2027-09-01**, on unseen MCAT-style passages, students who passed the held-out deep-structure sort beat students **matched on time, baseline ability, and perturbed-item accuracy** but *not* sort-verified by **≥0.3 d**. Non-retreatable: if the two matched groups are equal on unseen transfer, the sort gate adds nothing ("feels deeper" doesn't count).

How to resolve it: A pre-registered volume-matched study with the sort items held out, baseline ability + perturbed accuracy matched across arms; outcome = unseen MCAT-style passage application.

Validation: Test 1 — disputed (GPT 6/dispute, Claude dispute) → PASS. Test 2 — crux PASS, retreat-resistance PASS, reach PASS (a chess/coding-pattern trainer: a held-out principle-sort predicts transfer better than studied-item accuracy). Test 3 — **HOLDS-PENDING:** Chi deep-structure + transfer-appropriate processing are established; the ≥0.3 d *incremental-prediction* delta pends held-out data.

**Spiky POV 3 (Strong) — Build a curated library of human-reviewed discrimination drills for known MCAT confusable sets, NOT a generic interleave toggle.** Target the specific confusions that sink MCAT students (acids/bases vs buffers; competitive / non-competitive / uncompetitive inhibition; SN1/SN2), each drill forcing the **discriminating mechanism + a differing consequence** — not the category label, not auto-tagged, and not for CARS/verbal.

Elaboration: The feature is a curated **concept-discrimination engine**, not Anki's syntactic randomization. A human-reviewed confusable-set library (or a curated concept graph) is built **after** baseline competence; each drill makes the student name *why* this case is X-not-Y (the mechanism) and *what observable consequence* differs. On the substrate this is **desktop-prepare** (the drill generation + concept graph live where the collection is visible) → **cross-platform review** (template JS), again per SPOV 4; auto-tagging is rejected because it re-imports faulty-AI risk. **Non-redundant:** AnKing tags don't force the discriminating mechanism, UWorld questions aren't curated confusable-pair drills, and Jack Westin is CARS-focused. Evidence: interleaving helps **most for confusable, similar categories** via discriminative contrast (Brunmair & Richter g=0.42 overall; math 0.34; **words −0.39, expository ns**) and Rohrer/Taylor show the *mechanism* (do not import math d=0.83 as the expected MCAT magnitude). Expert tension: disagrees with Anki-forum generic-interleaving and auto-tagging vendors; extends Brunmair & Richter + Rohrer/Taylor.

Prediction (or Disconfirmer): A volume-matched RCT by **2027-09-01**: curated mechanism-discrimination drills beat **both** a high-quality *random interleave on the same concept graph* and a blocked arm on unseen confusable-pair science items by **≥0.3 d**, **AND** show ≈0 advantage on expository/verbal (consistent with words −0.39). Non-retreatable: if high-quality random interleave matches at equal volume, "curation beats shuffle" fails; if the drills help CARS/verbal, the science-only scoping was wrong.

How to resolve it: A 3-arm volume-matched RCT (curated drills vs high-quality random interleave on the *same* concept graph vs blocked), outcome = unseen confusable-pair items + an expository/verbal control set.

Validation: Test 1 — disputed (GPT 6/split, Claude dispute) → PASS. Test 2 — crux PASS, retreat-resistance PASS (compared against *high-quality* random interleave, not a straw shuffle), reach PASS (radiology/derm confusable-lesion-pair drills). Test 3 — **HOLDS-PENDING:** discriminative-contrast + "interleaving helps most for confusable similar categories" established; the 3-arm delta and the contested verbal boundary pend.

**Spiky POV 10 (Strong) — Difficulty must FADE SUPPORT within a per-concept-family expertise estimate, NOT offer a global "hard mode," and NOT apply to declarative recall.** Fade scaffolds only after ≥2 unaided successes + 1 transfer item in a family, fading *toward explaining the rejected alternative*; escalating difficulty on novices or on rote facts backfires (expertise reversal).

Elaboration: The feature is a **support-fading ladder per concept-family** (worked example → partial completion → open prediction → passage discrimination), driven by the *same* per-family expertise estimator SPOV 3 needs. It deliberately refuses a global difficulty slider and refuses to touch declarative recall (amino-acid properties, psych terms). On the substrate, the expertise estimate + fade thresholds are **desktop-prepared** state; the review tier reveals the next rung. **Non-redundant:** none of AnKing/UWorld/Jack Westin fade scaffolds per concept-family by a demonstrated-mastery estimate; they offer fixed cards or global question difficulty. Evidence (the **best-grounded** of the Strong tier): the **expertise-reversal effect is verbatim** (Tetzlaff 2025 d=0.505 for low-prior-knowledge / **−0.428** for high — 176 effect sizes), with Sweller/Chen/Kalyuga (high element interactivity) and Atkinson/Renkl (fading + principle prompts → transfer at no extra time). Expert tension: disagrees with "hard mode"/global-difficulty designs; extends Sweller/Kalyuga + Atkinson/Renkl.

Prediction (or Disconfirmer): Volume-matched by **2027-09-01**: per-family support-fading beats **both** a global-hard-mode arm and a no-fading arm on unseen high-element-interactivity items by **≥0.3 d**, **AND** shows ≈0/negative on declarative-recall families. Non-retreatable: if global hard-mode matches per-family fading on unseen high-element items, "fade don't escalate" fails; if fading also helps declarative recall, the high-element scoping was wrong.

How to resolve it: A volume-matched RCT (per-family fading vs global hard-mode vs no-fading) on high-element-interactivity items, with a declarative-recall control family to test the scoping boundary.

Validation: Test 1 — disputed (GPT 7/split, Claude split) → PASS. Test 2 — crux PASS (names the winning condition *and* its scoping disconfirmer), retreat-resistance PASS, reach PASS (math tutoring: per-skill fading > global slider; nothing for arithmetic fluency). Test 3 — **HOLDS-PENDING (best-grounded):** expertise reversal is established verbatim; the MCAT fading-vs-hard-mode delta pends, and ERE is "less clear" for humanities/verbal.

### WEAK — kept + flagged (Test 3 partly-supported/contested, reach weak, or Test 2 retreat-resistance required the adopted hardening)

**Spiky POV 1 (Weak) — The flagship card feature is a student-authored "miss → card" scaffold whose load-bearing required field is a DISCONFIRMER ("what one fact would flip this answer?") — NOT AI card-generation.** The tool perturbs the surface, prompts the underlying principle, flags traps, and requires a conjecture-first card with a disconfirmer field; it must **not** auto-generate the card body, because automation forfeits the generation effect *and* ships faulty LLM items.

Elaboration: On a missed AAMC/UWorld item, the scaffold walks the student to author a card that preserves the principle, varies the surface, and records **what one fact would flip the answer** (the disconfirmer / boundary case) — the student still produces the card. On the substrate this is a card-template + desktop-prepare authoring flow (no engine fork needed beyond storing the extra field). **Non-redundant:** AnKing/UWorld are premade decks/QBanks and Jack Westin doesn't turn *your* miss into a disconfirmer-bearing card. Evidence (Test-3 wording FIX applied — mechanism cites several primaries, not one): user-generated > premade (Pan 2023, d=0.45 memory / 0.29 application), the generation effect broadly (Bertsch 2007 d≈0.40; a 2023 text-generation meta g≈0.41 [meta-analysis]), and "all faulty" LLM medical items (BMC review) — automating the body would both lose generation and import faulty items. Expert tension: disagrees with Jarrett Ye-style LLM card-gen and UWorld UAsk / RemNote / AnkiHub Smart Search; extends Jack Westin's miss-review workflow + Pan/Agarwal.

Prediction (or Disconfirmer): A volume- and time-matched study by **2027-09-01** isolating the field: a **student-authored + disconfirmer** arm beats (a) **student-authored + ordinary explanation**, (b) **AI-draft + student-edit**, and (c) **premade cloze** on unseen MCAT-style transfer items by **d≥0.3**, **AND** the AI-auto arm shows ≥10% more faulty cards on blind audit. Non-retreatable: if disconfirmer-authoring doesn't beat *student-authored-with-ordinary-explanation* at equal time, the *disconfirmer field* (not just "authoring") earns nothing ("liked it"/"made more cards" doesn't count).

How to resolve it: The 4-arm equal-time study above (isolating the disconfirmer field from generic authoring and from AI drafting), outcome = unseen transfer items + a blind faulty-card audit.

Validation: Test 1 — disputed (GPT 7/split, Claude dispute/split; spike = anti-AI + the disconfirmer requirement) → PASS. Test 2 — crux WEAK→**fixed** by isolating disconfirmer vs ordinary-explanation vs AI-draft-edit vs premade at equal time; reach PASS (USMLE Step 1 / bar prep). Test 3 — **HOLDS-PENDING:** the generation effect is established; the disconfirmer-*specific* delta is untested and generation>premade is load-moderated (can reverse under high load).

**Spiky POV 5 (Weak) — Instrument the fork's own learning theory with item DISCRIMINATION + a Preparation-for-Future-Learning proxy, and REFUSE perturbed-accuracy / streaks / deck-completion % as EVIDENCE OF LEARNING.** Build empirical item discrimination (point-biserial) + a PFL proxy ("after remediation, fewer trials to acquire the NEXT concept in the cluster") into the product, and refuse to report perturbed-item accuracy, streaks, or completion % *as proof a feature worked* (they remain fine as ordinary UX/ops metrics).

Elaboration: This is the **falsifiability layer** that stops "learning-science features" from being unfalsifiable UX decoration: the fork computes whether "transfer-trained" items actually *discriminate* better than plain recall clozes, and whether remediation buys faster acquisition of the next concept. On the substrate, discrimination + PFL are **desktop-prepared analytics** over synced review logs. **Non-redundant:** none of AnKing/UWorld/Jack Westin instrument their *own* theory — they report % correct and streaks. Evidence: PFL is canonical (Bransford & Schwartz), and the precise mechanism (Test-3 FIX) is **Williams/Lombrozo: explanation can drive *overgeneralization* / illusory patterns** — i.e. rising accuracy can reflect a surface shortcut, so accuracy is the wrong sole KPI. Expert tension: extends Bransford & Schwartz + Williams/Lombrozo; disagrees with heatmap/streak culture.

Prediction (or Disconfirmer): By **2027-09-01**, "transfer-trained" items show **Δ point-biserial ≥0.1** over plain recall clozes **AND** remediated clusters show fewer trials-to-next-concept than matched non-remediated clusters; **AND** discrimination + PFL predict unseen transfer (or detect a deliberately broken feature) better than perturbed-accuracy, streaks, or completion %. Non-retreatable: if "transfer-trained" items discriminate no better than recall clozes, the transfer theory is falsified by its own metric (rising accuracy/streaks don't rescue it).

How to resolve it: Compute point-biserial discrimination + the trials-to-next-concept PFL proxy on held-out data; run a planted-broken-feature detection test comparing these metrics against accuracy/streaks/completion.

Validation: Test 1 — disputed (GPT 7/split, Claude split) → PASS. Test 2 — crux WEAK→**fixed** by scoping "refuse as **evidence of learning**" (still allow as UX/ops); reach PASS (corporate-compliance / coding-bootcamp LMS claiming "deep learning"). Test 3 — **HOLDS-PENDING:** PFL canonical and the explanation→overgeneralization mechanism supported (looser mapping than "accuracy is gameable"); the comparative predictive claim pends.

**Spiky POV 6 (Weak) — MC/application cards are unreliable and false-knowledge-prone UNLESS wrapped in a feedback + error-classification engine; the engine is the feature, not more questions.** Gate every MC/application card behind: commit an answer → **classify the error** (content / reasoning / trap / timing) against source-backed distractor rationales → queue **one** targeted remediation card. The buildable value is the engine; bare MC encodes lures and the alternative-accessibility problem.

Elaboration: The feature is a **distractor-feedback + forced-error-classification loop** that ends in *one* (load-managed, not over-atomized) remediation card scheduled into FSRS. On the substrate this is a card-template interaction (commit-then-reveal + a classification step) plus a desktop-prepare remediation generator. **Non-redundant:** UWorld shows rationales but doesn't force *error-classification* and doesn't queue one targeted remediation card into your SRS; AnKing has neither. Evidence: MC creates false knowledge via lures, and **feedback** reduces lure intrusions / doubles low-confidence-correct retention (Butler & Roediger 2008); retrieval is **near-zero without feedback at ≤50% accuracy** (Rowland 2014, g≈0.03; verified k=17, CI spans 0). Expert tension: disagrees with "just do more UWorld"/screenshot cards; extends Butler & Roediger + Jack Westin's error taxonomy.

Prediction (or Disconfirmer): A volume-matched RCT by **2027-09-01**: MC **without** distractor feedback + forced error-classification produces **≤0** gain on unseen items vs no-MC practice (for low-accuracy/no-feedback regimes); the **same cards with the engine** produce **≥0.3 d**. Non-retreatable: if bare MC matches engine-wrapped MC at equal volume, "the engine is the feature" fails ("did more questions" doesn't count).

How to resolve it: A volume-matched RCT (no-MC vs bare-MC vs engine-wrapped MC), outcome = unseen MCAT-style items; report the engine's *incremental* lift.

Validation: Test 1 — disputed (GPT 6/dispute, Claude dispute; spike = "unreliable unless wrapped") → PASS. Test 2 — crux WEAK→**fixed** by softening "net-negative" → "unreliable/false-knowledge-prone unless wrapped" and leading with the engine's incremental lift; reach PASS (bar / NCLEX MC trainer). Test 3 — **HOLDS-PENDING:** Butler & Roediger lures + Rowland g≈0.03 (no-feedback ≤50%) both verified ("≤0 without feedback" holds for low-accuracy/no-feedback, not universally); the engine's ≥0.3 d pends.

**Spiky POV 7 (Weak) — Capture confidence ONLY after a committed answer and use it ONLY to route HIGH-CONFIDENCE-WRONG to a misconception drill; refuse a vanity confidence dashboard and a confidence-only FSRS interval change.** Record which false model drove the high-confidence error and route it to a source-backed misconception drill. Do **not** show confidence as a dashboard metric, and do **not** feed confidence into FSRS intervals **unless** doing so beats routing on *unassisted* items — both are Goodhart traps.

Elaboration: The single defensible use of confidence is the **high-confidence-wrong → misconception-drill router** (capturing the false model). On the substrate, confidence is captured at answer-time (template JS) and routing/drill-generation is desktop-prepared. **Non-redundant:** none of AnKing/UWorld/Jack Westin capture committed-answer confidence and route the dangerous quadrant to targeted remediation. Evidence: JOLs are biased (foresight bias; cue-dependency — Koriat & Bjork 2005), so a confidence *dashboard* invites illusion-of-competence and hedging. **Test-3 FIX: the Lee 2025 "calibration training +8.9%/+4.1%" result is thin/unverified-this-pass and is explicitly NOT load-bearing here.** Expert tension: disagrees with calibration-dashboard / predict-your-score products; extends Nelson & Narens / Koriat + the parent's "scheduler ≠ scorer" separation.

Prediction (or Disconfirmer): By **2027-09-01**, routing high-confidence-wrong → targeted drills reduces repeat high-confidence-wrong errors on **unseen same-family items by ≥20%** vs capture-but-don't-route; **AND** a confidence-driven-FSRS variant shows no recall benefit (and a confidence dashboard yields no unassisted gain). Non-retreatable: if routing doesn't beat capture-only on *unassisted* items, confidence earns no place; if a confidence feature *does* beat routing on unassisted items, the "no dashboard / no FSRS input" rule is falsified there.

How to resolve it: An RCT (capture-only vs capture+route vs confidence-driven-FSRS vs confidence-dashboard), outcome = unassisted unseen same-family error rate + recall.

Validation: Test 1 — disputed (GPT 5/dispute, Claude dispute) → PASS. Test 2 — crux WEAK→**fixed** by scoping to "no **vanity** dashboard + no confidence-only interval change **unless** it beats routing on unassisted items"; reach PASS (pilot-training / medical-CME high-confidence-error tool). Test 3 — **HOLDS-PENDING:** JOL-bias / foresight-bias established; the "beyond-routing fails" claim is largely untested.

**Spiky POV 8 (Weak — PARTLY-CONTESTED) — Ship only NON-DISTORTING motivation; categorically exclude the SPECIFIC mechanics that depress error-confrontation (public leaderboards/ranking, loss-framed streaks) — even though gamification metas are net-positive on average.** Allow private mastery/coverage, recovery streaks that reward *returning after a lapse*, and effort-credit for hard/low-confidence items; ban the specific mechanics that measurably reduce hard-card attempts, low-confidence admissions, or remediation time.

Elaboration: The rule is **mechanism-level, not class-level**: the fork keeps motivation that rewards the exact behaviors MCAT gains require (confronting errors, doing hard reviews) and bans the ones that punish them. On the substrate this is pure product/UX layered over the engine. **Non-redundant:** generic study apps bolt on streaks/leaderboards; this is the explicit decision to exclude the distorting subset. ## DOK 3: Insights

The 12 validated DOK 3 insights (after isolated cross-family critique + revision), grouped by theme. Each cites the facts it rests on, and is tagged **MEASUREMENT-strong** (grounded in established effects), **TESTABLE-HYPOTHESIS** (carries a resolver; asserted as a bet), or **BUILDABILITY-CONSTRAINT** (shapes the "how" / gates cross-platform claims). Cross-cutting frame: "better Anki features → MCAT gains" is **OBSERVATIONAL** (Anki↔exam non-randomized; the one applied anchor — Wothe Step-2-CK — is null), so every transfer/application claim is a testable hypothesis whose resolver is a volume-matched RCT on unseen MCAT-style items.

### Build the missing learning layer (not another flashcard shell)

**Insight 1 (TESTABLE-HYPOTHESIS / MEASUREMENT-strong):** The highest-leverage bet is **not** AI card generation; it is a **student-driven miss-review + authoring scaffold that makes explanations hard to vary.** The tool perturbs the surface, prompts the underlying principle, flags traps, and asks for a conjecture-first card whose load-bearing field is a **DISCONFIRMER** ("what one fact would flip this answer?"); the student still produces the card (automating the body forfeits the generation advantage *and* collapses into the faulty AI card-gen the corpus rejects). Strongest version adds a boundary case against the student's stated principle. *Rests on:* Pan 2023 (d=0.45/0.29); Bertsch 2007 (~.40) + a 2023 text-generation meta (g≈0.41); Rowland 2014; BMC "all faulty"; parent transfer moderators + parent SPOV 11 WEAK; cogsci hard-to-vary/disconfirmer.

**Insight 2 (BUILDABILITY-CONSTRAINT / TESTABLE-HYPOTHESIS):** "More reviews" is a false lever **only for APPLICATION/transfer, not for memory.** Volume/spacing/adaptive scheduling DO move surface memory (don't pretend FSRS is broken); the feature-layer delta is narrower — exam-horizon desired-retention ramps, application queues, failure-triggered remediation layered over FSRS. True queue **re-ranking** for concept/application priority is fork-only; Anki custom scheduling re-weights a per-card next interval, it cannot reorder the queue from semantic context. *Rests on:* Donoghue & Hattie d=0.85/0.74; Eglington & Pavlik +40%; Lindsey +16.5%/+10%; FSRS Helper/FSRS-6/per-deck DR; custom-scheduler context `{deck_name, seed}` + per-card re-weight limit; parent Insight 7.

### Transfer has to be engineered, narrowly

**Insight 3 (TESTABLE-HYPOTHESIS / MEASUREMENT-strong):** Application cards are only a testable transfer hypothesis if they **preserve deep structure while perturbing surface cues AND THEN verify representation shift.** Near discrimination among confusable science concepts is defensible; passage-level reasoning transfer is contested. Pair surface-varied minivignettes with a **deep-structure sort probe** ("which two share a principle?"); perturbed-item accuracy alone is insufficient (students learn incidental shortcuts). *Rests on:* parent transfer d=0.40 + moderators / far ~0; parent SPOV 11 WEAK (words −0.39, expository ns); Agarwal transfer-appropriate processing; Koriat/Bjork cue-dependency; Williams/Lombrozo shortcut risk [critique-sourced].

**Insight 4 (TESTABLE-HYPOTHESIS / BUILDABILITY-CONSTRAINT — strongest feature-novel):** The strongest novel feature is **CURATED discrimination drills for known MCAT confusions, not generic interleaving.** Use human-reviewed confusable sets / a curated concept graph (NOT auto-tagging, which re-imports faulty-AI risk), generated after baseline competence; each drill forces naming the discriminating **mechanism** + a falsifiable consequence. Mechanism plausibly transfers to confusable science clusters; magnitude untested for MCAT (don't import Rohrer math d=0.83 as the expected effect). *Rests on:* Brunmair & Richter (math +, words −0.39, expository ns); Rohrer/Taylor as mechanism not MCAT magnitude; learner blocking preference; Anki lacks semantic-topic interleaving; engineer: review-interleaving is fork-only (desktop-prepare drill gen + cross-platform review).

**Insight 5 (BUILDABILITY-CONSTRAINT, inherited):** CARS forces the product to decide whether it is a **flashcard fork or a broader MCAT practice system.** Recall offers **no diagnostic signal** for the reasoning/timing/tone/argument failures that limit CARS; the real feature decision is build a separate passage/reasoning module (with error-pattern tracking) vs explicitly leave CARS to integrations (Jack Westin) and keep the fork science-content-focused. *Rests on:* parent CARS no-content + section-aware; practitioner consensus; Pan far-transfer boundaries.

### Difficulty & feedback

**Insight 6 (TESTABLE-HYPOTHESIS / MEASUREMENT-strong):** Difficulty should **FADE SUPPORT inside validated concept families, not globally escalate to "hard mode."** For problem-solving / high-element-interactivity content, fade only after ≥2 unaided successes + 1 transfer item within a family; fade **toward explaining the rejected alternatives** / counterfactual branches. Needs the same per-concept expertise estimator as Insight 4; weak fit for declarative recall. *Rests on:* Tetzlaff 2025 (d=0.505 / −0.428); Sweller/Chen/Kalyuga; Atkinson/Renkl fading + principle prompts; Karpicke 2025; cogsci counterfactual/rules-out.

**Insight 7 (MEASUREMENT-strong / TESTABLE-HYPOTHESIS):** Multiple-choice practice can be **net-NEGATIVE without distractor-level feedback + remediation.** Not "every card needs an explanation" (unscaffolded self-explanation is null/harmful); the distinctive feature = feedback that forces **error-classification** + rewrite/remediation with **source-backed distractor rationales** (why the lure was tempting, what model it exposes, what replacement drill follows) — while managing load (over-atomized remediation = review debt). *Rests on:* Butler & Roediger lures; Butler/Karpicke/Roediger feedback; Rowland g=0.03 ≤50%; McCarthy/iSTART null/detrimental; Kluger & DeNisi (~⅓ hurt) [critique-sourced]; med-ed vignette critiques.

### Calibration & motivation (design constraints, not dashboards)

**Insight 8 (MEASUREMENT-strong — cleanest pass):** Calibration is valuable **only when it captures committed-answer mismatches and routes high-confidence-wrong to misconception remediation.** Score confidence ONLY after a committed answer (don't reward hedging — Goodhart guard); record which false model drove the high-confidence error; the jewel = high-confidence-wrong → source-backed misconception drill; never use confidence alone to change FSRS intervals. The mismatch-routing taxonomy is an untested operational hypothesis. *Rests on:* Nelson & Narens; Koriat JOL biases; Lee 2025 +8.9%/+4.1% [thin — not load-bearing alone]; Butler feedback; parent score-honesty separation; cogsci Goodhart guard.

**Insight 9 (TESTABLE-HYPOTHESIS):** The problem with gamification is **MECHANISM DISTORTION, not "gamification is bad."** Net-positive gamification metas mean don't reject the class; the narrow claim: engagement/competition/streak-optimizing mechanics can **punish** the behaviors MCAT gains require (confronting errors, hard reviews, admitting low confidence, remediation time) — and net-positive metas don't rebut it (they measure engagement/short-term/surface, not error-confrontation/transfer). Use non-distorting gamification (private mastery/coverage, recovery streaks, effort-credit for hard items, verbal encouragement); no public ranking / streak-loss penalty. *Rests on:* Sailer & Homner 2020 g≈0.49 / Huang 2020 g≈.464 / Frontiers 2023 g≈.82 [critique-sourced]; Deci 1999 (tangible-reward caveat, NOT all gamification); Almeida 2023; Huang 2024 LBL-negative; streak-anxiety.

### AI's safe lane is scaffolding, not authority

**Insight 10 (TESTABLE-HYPOTHESIS, inherited):** AI should **GENERATE severe tests against a human-authored key, not CERTIFY understanding.** AI can produce counterexamples/boundary cases/contradiction checks/disconfirmer prompts, but must NOT grade explanation quality as authoritative (resemblance-to-key is Goodhartable). Test the AI lane against the **crutch effect**: if assisted performance rises while unassisted MCAT-style performance falls, the feature fails even if students like it (the MCAT is taken unassisted). Pro-AI evidence is thin/preprint. *Rests on:* Chen 2026 +11.9pp (MC ns) [preprint]; LearnLM/Eedi +5.5pp / 76.4% [preprint]; BMC "all faulty"; GPT-4 +48% assisted / −17% unassisted (Bastani et al. 2024, PNAS) [critique-sourced]; SmartFlash editability; parent AI-lane separation.

### Metrics & architecture (make the theory falsifiable; obey the substrate)

**Insight 11 (MEASUREMENT-strong — metric layer):** The fork should make its **own theory of learning falsifiable** with discrimination + Preparation-for-Future-Learning metrics. Perturbed-item accuracy is too weak a KPI (gameable by surface shortcuts; capped at sequestered problem-solving); better transfer KPIs = empirical item **discrimination**, boundary/competing-principle item performance, and a **PFL proxy** ("after remediation, fewer trials to acquire the next concept in the cluster"). This is what stops "learning-science features" from being unfalsifiable UX decoration. *Rests on:* Bransford & Schwartz PFL [critique-sourced]; Williams/Lombrozo explanation→overgeneralization [critique-sourced]; parent transfer boundaries; Insight 3 deep-structure probe; Insight 4 clusters.

**Insight 12 (BUILDABILITY-CONSTRAINT — architecture):** Buildability forces a **TWO-TIER architecture — desktop "prepare," cross-platform "review."** The desktop prepare tier (Python/Rust, sees the whole collection) emits small syncable artifacts (tags / a companion store); the review tier is card-template JavaScript that works on AnkiMobile + AnkiDroid. Mandatory because: `customData` <100 bytes / 8-byte keys; custom scheduling sees only `{deck_name, seed}` + re-weights a per-card next interval (NOT a queue re-rank); AnkiMobile no add-ons / AnkiDroid no Python. True concept-interleaving + semantic queue re-ranking are **fork-only**; front-side incremental cloze is a **template** problem (Anki-Cloze-Interactive), not a scheduler feature. *Rests on:* engineer code-verified Anki constraints (`scheduler.proto`, `rslib/data.rs`, v3 FAQ).

### C. Transfer & interleaving features (+ boundaries)
- **Source — Conditional transfer (parent BrainLift: mcat-anki-study-app; Pan & Rickard 2018; Agarwal).** *DOK 1 facts:* testing→transfer **d=0.40** (0.58 with response congruency, 0.28 without, ~0 after bias-correction when moderators absent); far transfer ~0; transfer-appropriate processing (match quiz↔test type). *DOK 2 summary:* transfer isn't free — build the moderators into the card/task type. *Link to source:* https://doi.org/10.1037/bul0000151
- **Source — Interleaving & contextual interference (Brunmair & Richter 2019; Rohrer 2020; Taylor & Rohrer 2010; Battig; Shea & Morgan).** *DOK 1 facts:* interleaving g=0.42 overall; paintings .67, math .34, **words −0.39, expository ns**; works for **confusable categories** via discriminative contrast; interleaved math d=0.83 / d=1.21; contextual interference helps retention/transfer but hurts acquisition. *DOK 2 summary:* concept-aware interleaving for **confusable science/quant**; NOT words/verbal/expository, not forced before success — and use Rohrer as **mechanism, not MCAT magnitude.** *Link to source:* https://doi.org/10.1037/bul0000209 ; http://uweb.cas.usf.edu/~drohrer/
- **Source — CARS (parent BrainLift; AAMC; practitioners).** *DOK 1 facts:* **CARS requires no specific content knowledge** (AAMC); practitioners: Anki is the wrong tool for CARS, which needs daily passages + reasoning/error-pattern tracking. *DOK 2 summary:* CARS = passage/reasoning module, not content cards. *Link to source:* https://jackwestin.com/cars

### D. Elaboration / self-explanation / dual-coding features
- **Source — Self-explanation & worked-example fading (Chi 1989; Bisra 2018; Atkinson/Renkl 2003; Rittle-Johnson 2017; McCarthy et al. 2018 — FIX #2).** *DOK 1 facts:* good self-explainers link actions→principles + monitor (Chi); prompted self-explanation **g=.55** (Bisra 2018); **fading + principle prompts → near+far transfer, no extra time** (Atkinson/Renkl); BUT self-explanation can reduce detail recall / overgeneralize / be null if methods wrong (Rittle-Johnson); **iSTART metacognitive prompts null/detrimental, worst for weak readers** (McCarthy et al. 2018). *DOK 2 summary:* self-explanation is a top candidate **only if scaffolded** (principle-directed prompts + examples + fading + feedback). *Link to source:* https://doi.org/10.1207/s15516709cog1302_1 ; https://doi.org/10.1007/s10648-018-9434-x ; https://doi.org/10.1037/0022-0663.95.4.774
- **Source — Dual coding / multimedia (Paivio; Mayer; Rey 2012; Backhouse; Moro; Styn).** *DOK 1 facts:* dual coding; multimedia = dual channels / limited capacity / active integration; **seductive details −0.30 retention / −0.48 transfer** (Rey 2012); drawing histology improved retention; image-SBA > text. *DOK 2 summary:* image/drawing justified for **spatial/pathway material, not decoration**; Image Occlusion already exists → the gap is guided drawing/labeling + image-transfer prompts + coherence checks. *Link to source:* https://doi.org/10.1016/j.edurev.2012.05.003

### E. Spacing & scheduling features
- **Source — Spacing & adaptive scheduling (Cepeda 2008; Donoghue & Hattie 2021; Eglington & Pavlik 2020; Lindsey 2014; Kerfoot 2007/2009/2010).** *DOK 1 facts:* optimal gap scales with retention horizon; distributed practice **d=0.85** / testing **d=0.74** (mostly surface outcomes); **adaptive item scheduling up to 40% more recalled**; personalized review **+16.5% / +10%**; spaced medical cases d=1.01 + transfer of image-diagnosis + 2-yr retention. *DOK 2 summary:* spacing is real but Anki is already strong → the gap is **exam-horizon semantics** (MCAT date, section priority, DR ramp, case/image/application queues respecting FSRS). *Link to source:* https://doi.org/10.1111/j.1467-9280.2008.02209.x ; https://doi.org/10.3389/feduc.2021.581216 ; https://doi.org/10.1038/s41539-020-00074-4 ; https://doi.org/10.1177/0956797613504302 ; https://pubmed.ncbi.nlm.nih.gov/?term=Kerfoot+spaced+education
- **Source — Exam-date scheduling reality (Anki forums; FSRS tutorial).** *DOK 1 facts:* exam-date/deadline = a common forum request with **NO native feature** (workarounds: filtered decks / max-interval); **DR >90% workload rises fast, >97% overwhelming**; Hard-misuse corrupts FSRS; scheduler↔scorer feedback loop (parent). *DOK 2 summary:* the right request is an **exam-date-aware FSRS policy + workload caps**, NOT a naive cram override. *Link to source:* https://forums.ankiweb.net/

### F. Metacognition & feedback features
- **Source — Metacognition (Nelson & Narens 1990; Koriat 1997; Lee 2025 [preprint, thin]; Butler 2008).** *DOK 1 facts:* object/meta levels; **JOLs discount repetition/duration vs actual recall**; practice impairs calibration (underconfidence) but improves resolution; AI calibration RCT +8.9% learning / +4.1% calibration (Lee 2025) **[thin — down-weighted, not load-bearing]**; feedback corrects memory + metacognitive errors. *DOK 2 summary:* confidence capture is valuable only as **calibration training + feedback / high-confidence-wrong routing**, not a scheduling oracle or a vanity dashboard. *Link to source:* https://doi.org/10.1037/0096-3445.126.4.349

### G. Motivation / gamification (mostly cautionary)
- **Source — Motivation primaries (Deci 1999; Almeida 2023; Huang 2024; Waterloo 2026; + net-positive metas: Sailer & Homner 2020; Huang 2020; Frontiers 2023 [critique-sourced]).** *DOK 1 facts:* expected tangible rewards undermine intrinsic motivation (engagement −0.40 / completion −0.36 / performance −0.28), positive verbal feedback +0.31/+0.33 (Deci 1999); 87 negative-gamification papers, esp. badges/leaderboards/competition/points (Almeida 2023); **Levels+Badges+Leaderboards negative + shames low performers** (Huang 2024); streak-anxiety / "not losing the streak" → avoid hard lessons; **BUT gamification net-positive on average (Sailer & Homner g≈0.49; Huang 2020 g≈.464; Frontiers 2023 g≈.82).** *DOK 2 summary:* safest motivation = **mastery/coverage/verbal encouragement/load-relief**; the net-positive metas are the named contested point for SPOV 8 (they measure engagement, not error-confrontation). *Link to source:* https://doi.org/10.1037/0033-2909.125.6.627 ; https://selfdeterminationtheory.org/ ; https://doi.org/10.1007/s10648-019-09498-w
- **Source — Cognitive-load & overload (Mayer; Anki leech/backlog docs; choice-overload lit [mixed]).** *DOK 1 facts:* choice overload reduces action under complexity/anxiety; Anki reviews compound (leeches/backlog); over-atomization violates the Minimum Information Principle. *DOK 2 summary:* feature bloat isn't neutral → ship **guided workflows, not a cockpit of knobs.** *Link to source:* https://docs.ankiweb.net/leeches.html

### H. AI-assisted study features
- **Source — Pro-AI evidence [preprint] (Chen 2026; LearnLM/Eedi 2025; SmartFlash; existing AI add-ons).** *DOK 1 facts:* LLM open-ended self-explanation **+11.9pp** transfer-explanation (MC accuracy ns; ¼ as many problems) [preprint]; LearnLM/Eedi expert-supervised drafts **76.4%** minimal edits, +5.5pp novel problems [preprint]; SmartFlash users want editable/transparent AI (cognitive ownership); existing add-ons (MyAnswerChecker semantic answer-checking, AnkiAIUtils failure-triggered elaboration, Anki AI Explainer, Course Tutor MCP). *DOK 2 summary:* AI's defensible lane = **retrieval/elaboration scaffold + severe-test generation**, NOT silent score-bearing authoring or final grading. *Link to source:* https://arxiv.org/abs/2512.23633
- **Source — Anti-AI / risk evidence (BMC review 2024; Bastani et al. 2024 PNAS; pseudo-vignettes 2024; vendor claims).** *DOK 1 facts:* BMC review found LLM medical items **"all faulty"**; "faster completion, less learning" — **GPT-4 +48% assisted / −17% UNASSISTED** (Bastani et al. 2024) [critique-sourced]; UWorld UAsk / RemNote / AnkiHub Smart Search = **vendor claims, not outcomes**. *DOK 2 summary:* efficacy is thin and risks are asymmetric → **quarantine AI content from grading** unless grounded/editable/reviewed/validated; kill any assisted-up/unassisted-down scaffold. *Link to source:* https://doi.org/10.1186/s12909-024-05601-0 ; https://papers.ssrn.com/sol3/papers.cfm?abstract_id=4895486

### I. The Anki↔exam evidence base (OBSERVATIONAL)
- **Source — Observational Anki↔exam studies (Frappa et al. 2026 — FIX #1; Deng 2015; Gilbert 2023; Winter 2025; Wothe et al. — FIX #4).** *DOK 1 facts:* **Frappa et al. 2026 review: 11 studies, NONE randomized** (Lu: users 241.1 vs 235.5, p=0.012); Deng 2015: unique cards predict Step 1 (B=5.9e-4, ~1pt/1,700 cards); Gilbert 2023: +6.4–12.9% (CBSE +12.9%, MCAT-adjusted); Winter 2025: ≥9,390 mature cards → 71.56% vs 60.05%; **Wothe et al. Step-2-CK null (252.5 vs 247.0, p=0.440).** *DOK 2 summary:* strong **ASSOCIATION, not causation** → building better MCAT features is a plausible bet, but "flashcards cause reasoning gains" must be **tested** (the applied/Step-2 evidence is mixed/null and non-randomized) — this is the domain ceiling under every transfer SPOV. *Link to source:* https://link.springer.com/journal/40670 ; https://doi.org/10.1007/s40037-015-0220-x
