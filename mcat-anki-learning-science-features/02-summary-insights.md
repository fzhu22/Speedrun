## DOK 2 - Knowledge Tree

### A. Anki implementation substrate & "already-built-in vs genuine-gap" map
- **Anki primitives/docs:** notes=fields, templates=blueprints, one note→many cards; Cloze special (no extra templates); Image Occlusion native since 23.10 (Hide-All/Hide-One); deck options already include new/review mixing, random gather/sort, sibling burying, Auto Advance, Easy Days, FSRS, review order by due/relative-overdueness/ascending-retrievability; custom scheduling = GLOBAL JavaScript over `states` (cross-platform but collection-wide); v3 scheduler can't be monkey-patched; rslib/fsrs-rs = source of truth (Python via rsbridge/protobuf); AnkiMobile no add-ons / AnkiDroid no Python add-ons (JS only); native clozes HTML-converted → front-side incremental cloze needs Closet/fork. **Summary:** the "obvious" features are built-in; the fork-worthy gaps are substrate-limited learning features (concept-aware sequencing, calibration capture, principle-directed self-explanation, front-side incremental cloze, exam-date/per-deck scheduling, error-driven remediation).
- **FSRS Helper / releases / FSRS-6-7:** Helper already does Postpone/Advance/Load Balance/Easy Days/Disperse Siblings/Flatten/True Retention; FSRS-6 in 25.07 (decay w20, 21 params, RMSE −5-6%); per-deck DR in 25.09; FSRS-7 emerging (21→35 params, fractional intervals, active regressions). **Summary:** the scheduler is the strongest part — don't reimplement FSRS; add MCAT policy layers it doesn't own.

### B. Retrieval / generation card-design features
- Generation > reading across test types (Slamecka & Graf 1978); generation effect ~.40 (Bertsch 2007); testing>restudy delayed, more-testing-not-study after one success (Roediger & Karpicke 2006/2008); testing g=0.50 weighted/0.54 unweighted, cued>free/recognition, near-zero (g=0.03) without feedback ≤50% (Rowland 2014); user-generated>premade d=0.45 memory/0.29 application (Pan 2023). **Summary:** evidence points to card CREATION + production, not consuming more premade clozes.
- MC lures create false knowledge; feedback reduces lure intrusions + corrects metacognitive errors, doubles low-confidence-correct retention (Butler & Roediger 2008; Butler/Karpicke/Roediger 2008); JOLs target-present → illusion of competence; retrieval cue-dependent (Koriat & Bjork 2005; Bjork/Dunlosky/Kornell 2013); med-ed critiques: isolated-fact/vignette-recognition undermines integration (2026/2023). **Summary:** MC cards valuable only as feedback-rich discrimination practice, not answer recognition.

### C. Transfer & interleaving features (+ boundaries)
- Parent: transfer conditional (Pan & Rickard d=0.40; moderators; ~0 without); far transfer ~0; Agarwal transfer-appropriate processing (match quiz↔test type). **Summary:** transfer isn't free; build moderators into the card/task type.
- Interleaving g=0.42 overall; paintings .67, math .34, words −0.39, expository ns; works for confusable categories (Brunmair & Richter 2019); interleaved math d=0.83 RCT / d=1.21 (Rohrer 2020; Taylor & Rohrer 2010); contextual interference helps retention/transfer but hurts acquisition (Battig; Shea & Morgan). **Summary:** concept-aware interleaving for confusable science/quant; NOT words/verbal/expository, not forced before success.
- CARS no content (parent); practitioners: Anki wrong tool for CARS, needs daily passages + reasoning/error-pattern tracking. **Summary:** CARS = passage/reasoning module, not content cards.

### D. Elaboration / self-explanation / dual-coding features
- Good self-explainers link actions→principles + monitor (Chi 1989); prompted self-explanation g=.55 (Bisra 2018, recommends computer-generated prompts); fading + principle prompts → near+far transfer, no extra time (Atkinson/Renkl 2003); BUT self-explanation can reduce detail recall / overgeneralize / be null if methods wrong (Rittle-Johnson 2017); iSTART metacognitive prompts null/detrimental, worst for weak readers (McCarthy et al. 2018). **Summary:** self-explanation is a top candidate ONLY if scaffolded (principle-directed prompts + examples + fading + feedback).
- Dual coding (Paivio); multimedia = dual channels/limited capacity/active integration, extraneous crowds out essential (Mayer); seductive details −0.30 retention/−0.48 transfer (Rey 2012); drawing histology improved retention 4/6wk, image-SBA>text (Backhouse; Moro; Styn). **Summary:** image/drawing justified for spatial/pathway material, NOT decoration; IO already exists → gap is guided drawing/labeling + image-transfer prompts + coherence checks.

### E. Spacing & scheduling features
- Optimal gap scales with retention horizon (Cepeda 2008); distributed practice d=0.85 / testing d=0.74 (Donoghue & Hattie 2021, mostly surface outcomes); adaptive scheduling +40% recalled (Eglington & Pavlik 2020); personalized review +16.5%/+10% (Lindsey 2014); spaced medical cases d=1.01 + transfer of image-diagnosis + 2-yr retention (Kerfoot 2007/2009/2010). **Summary:** spacing real but Anki already strong → gap is exam-horizon semantics (MCAT date, section priority, DR ramp, case/image/application queues respecting FSRS).
- Exam-date/deadline = common forum request, NO native feature (workarounds: filtered decks/max-interval); DR >90% workload rises fast, >97% overwhelming; Hard-misuse corrupts FSRS; scheduler↔scorer feedback loop (parent). **Summary:** the right request is exam-date-aware FSRS policy + workload caps, NOT a naive cram override.

### F. Metacognition & feedback features
- Object/meta levels (Nelson & Narens); JOLs discount repetition/duration vs actual recall (Koriat 1997); practice impairs calibration (underconfidence) but improves resolution; AI calibration RCT +8.9% learning/+4.1% calibration (Lee 2025); feedback corrects memory+metacognitive errors (Butler 2008). **Summary:** confidence capture valuable only as calibration TRAINING + feedback, not a scheduling oracle.

### G. Motivation / gamification (mostly cautionary)
- Expected tangible rewards undermine intrinsic motivation (engagement −0.40/completion −0.36/performance −0.28); positive verbal feedback +0.31/+0.33 (Deci 1999); 87 negative-gamification papers, esp badges/leaderboards/competition/points (Almeida 2023); Levels+Badges+Leaderboards negative + shames low performers (Huang 2024); streak-anxiety / "not losing the streak" / avoid hard lessons (Waterloo 2026). **Summary:** safest motivation = mastery/coverage/verbal encouragement/load-relief, NOT streaks/leaderboards.
- Choice overload reduces action under complexity/anxiety (mixed lit; JMIR 2025; NN/g); Anki reviews compound, leeches/backlog, over-atomization violates Minimum Information Principle. **Summary:** feature bloat isn't neutral → guided workflows, not a cockpit of knobs.

### H. AI-assisted study features
- LLM open-ended self-explanation +11.9pp transfer-explanation (MC accuracy ns; ¼ as many problems) (Chen 2026 [preprint]); LearnLM/Eedi expert-supervised drafts 76.4% minimal edits, +5.5pp novel problems (2025 [preprint]); SmartFlash users want editable/transparent AI (cognitive ownership); existing AI add-ons (MyAnswerChecker semantic answer-checking, AnkiAIUtils failure-triggered elaboration, Anki AI Explainer, Course Tutor MCP). **Summary:** AI's defensible lane = retrieval/elaboration scaffold (grade free responses, Socratic follow-ups, explain failures, help edit cards), NOT silent score-bearing authoring.
- BMC review "all faulty" LLM medical items; pseudo-vignettes (Algorithms 2024); "faster completion less learning" GPT-4 +48% assisted / −17% unassisted (2026 [preprint]); UWorld UAsk / RemNote / AnkiHub Smart Search = vendor claims not outcomes. **Summary:** efficacy thin, risks asymmetric → quarantine AI content from grading unless grounded/editable/reviewed/validated.

### I. The Anki↔exam evidence base (OBSERVATIONAL)
- Frappa et al. 2026 review: 11 studies, NONE randomized (Lu: users 241.1 vs 235.5, p=0.012; daily 238 vs 233.5, p=0.039); Deng 2015: unique cards predict Step 1 (B=5.9e-4, ~1pt/1,700 cards); Gilbert 2023: +6.4-12.9% (CBSE +12.9%, MCAT-adjusted); Winter 2025: ≥9,390 mature cards → 71.56% vs 60.05%; Wothe Step-2-CK null (252.5 vs 247.0, p=0.440). **Summary:** strong ASSOCIATION, not causation → building better MCAT features is a plausible bet, but "flashcards cause reasoning gains" must be tested (Step-2-applied evidence mixed/null, non-randomized).

## DOK 3 - Insights

### Theme 1 — Build the missing learning layer, not another flashcard shell
**Insight 1: The fork's killer feature is not "better Anki"; it is an MCAT-specific card-construction compiler that turns missed reasoning into future-transfer prompts.** Anki already handles spaced review, IO, type-in, Auto-Advance, filtered decks, randomization; the high-value gap is UPSTREAM — help students convert AAMC/UWorld misses into cards that preserve the principle, vary the surface, force generation, flag traps, and schedule a later application attempt. Strongest effects converge here: user-generated>premade (d=0.45/0.29), generation ~.40, cued production strong, transfer needs response-congruency/elaboration. *Rests on:* Anki built-ins/gaps (cartographer); Pan 2023; Bertsch 2007; Rowland 2014; parent transfer moderators; practitioner missed-Q→card workflows.

**Insight 2: A scheduler-first fork would optimize the part of Anki already closest to solved while leaving the MCAT bottleneck untouched.** FSRS+helpers solve recall-timing; the MCAT failure is "recognized the card, couldn't apply it." Treat FSRS as the memory substrate and build a POLICY layer (exam-horizon DR ramps, application-card queues, concept/discrimination re-ranking, failure-triggered remediation). "More reviews" is a false lever — application comes from generation/self-explanation/feedback/transfer-appropriate practice. *Rests on:* FSRS-6/substrate; Helper already does load-balancing/True-Retention; Eglington & Pavlik +40%; Lindsey +16.5%/+10%; Donoghue & Hattie (surface caveat); parent "scheduler must not become scorer."

### Theme 2 — Transfer has to be engineered, narrowly
**Insight 3: "Application cards" only deserve the name if they deliberately preserve the principle while perturbing the surface.** Not a cloze with a longer stem. Pan/Rickard moderators → a design rule: keep the response operation congruent with MCAT tasks, require elaborated retrieval, vary the scenario to break cue-dependency without far transfer. Build PAIRED cards: a minimal prerequisite-fact card + a surface-varied passage/minivignette card (predict / mechanism / trap discrimination). *Rests on:* parent transfer d=0.40 + moderators + far-transfer ~0; Agarwal transfer-appropriate processing; Koriat/Bjork cue-dependency; Jack Westin template; Rowland feedback/initial-success boundary.

**Insight 4: Interleaving should be a concept-discrimination engine, not a shuffle mode.** Anki's randomization is syntactic; it doesn't know which concepts are confusable. Interleave ONLY within section-bounded, concept-tagged science clusters (acids/bases vs buffers; inhibition types; enzyme kinetics; psych/soc theories), EXCLUDING CARS/verbal; surface the adherence cost honestly ("mixed practice feels worse now because it trains discrimination"). *Rests on:* Anki lacks semantic-topic interleaving; Brunmair & Richter (math .34, words −0.39, expository ns); Rohrer d=0.83 / Taylor & Rohrer d=1.21; 2024 learner blocking preference; parent CARS/interleaving heterogeneity.

**Insight 5: CARS is the boundary case that prevents the fork from becoming flashcard-maximalist.** If a feature works for biochem but is meaningless for CARS, the product learned the right lesson: MCAT studying is not one construct. CARS needs passage exposure + timing + tone/argument/question-type analysis + same-day reasoning review; a "CARS deck" launders a reasoning problem into a memory format and teaches false confidence. Build a separate passage module or leave CARS to integrations — don't pretend content cards cover it. *Rests on:* parent CARS no-content + section-aware insight; practitioner consensus; Pan far-transfer boundaries; Step-2-CK recognition critique.

### Theme 3 — Difficulty must adapt, not escalate
**Insight 6: The right difficulty feature is a support-fading system, not a "hard mode."** Sources jointly reject global difficulty-cranking. Novices need worked examples/hints/principle-prompts/feedback; advanced learners need less assistance + more generative application. Model expertise at the concept/card-family level and fade scaffolds: worked example → partial completion → open prediction → passage discrimination. *Rests on:* Tetzlaff 2025 (d=0.505 novices / −0.428 experts); Sweller/Chen/Kalyuga high-element-interactivity; Atkinson/Renkl fading+principle prompts; Karpicke 2025 (success with little support); Rowland no-feedback boundary.

**Insight 7: Feedback is not a nice-to-have for MCAT-style cards; it is the mechanism that makes multiple-choice practice safe.** MC matches exam format but introduces lures + can encode false knowledge. Every MC/application card should require an answer + confidence + explanation BEFORE reveal, then show targeted feedback (correct principle, why each distractor is tempting/wrong, what rewrite/remediation follows). Without this the fork amplifies the recognition-and-lure problem it should solve. *Rests on:* Butler & Roediger lures/feedback; Butler/Karpicke/Roediger feedback doubles low-confidence-correct; Rowland g=0.03 without feedback ≤50%; Bjork/Dunlosky/Kornell MC alternative accessibility; med-ed vignette-recognition critiques.

### Theme 4 — Calibration & motivation are design constraints, not dashboards
**Insight 8: Confidence capture becomes a learning feature only when paired with outcome feedback + remediation; otherwise it's just another illusion-of-competence field.** Raw JOLs are biased (discount repetition/duration, overtrust fluency), but predict-then-grade + feedback has evidence for learning + calibration. Capture confidence before reveal, compare to correctness + later retention, and route the mismatch: low-confidence-correct → consolidation; high-confidence-wrong → misconception remediation; repeated overconfidence → calibration coaching. Do NOT change FSRS intervals on confidence alone. *Rests on:* Nelson & Narens; Koriat JOL cue-utilization; Koriat/Bjork illusions; Lee 2025 (+8.9%/+4.1%); Butler feedback; parent metacognition/score-honesty separation.

**Insight 9: Gamification is the most tempting feature class to reject, because it optimizes visible engagement while risking the hidden variable that matters: willingness to do hard, honest practice.** Streaks/leaderboards/badges/levels make activity legible, but MCAT improvement depends on confronting errors, mixing hard concepts, tolerating low-confidence practice — behaviors status/reward systems can PUNISH (avoid hard reviews to preserve streaks/rank). Use mastery progress, coverage, verbal encouragement, load-relief — not public competition or streak loss-framing. *Rests on:* Deci overjustification (−0.40/−0.36/−0.28) + verbal feedback (+0.31/+0.33); Almeida 87 papers; Huang LBL-negative; streak-anxiety; Anki backlog/leech/overwhelm; choice overload.

### Theme 5 — AI's safe lane is scaffolding, not authority
**Insight 10: AI should grade and scaffold the student's retrieval, not replace it or silently author graded MCAT content.** Best AI evidence = self-explanation/tutoring-drafts/semantic answer-checking/failure-triggered elaboration; weakest/riskiest = auto-generated medical items + vendor flashcard conversion. Use AI AFTER the learner produces something: grade free responses, ask Socratic follow-ups, compare explanations to principles, suggest edits, draft editable+provenance-labeled remediation cards. Rule: AI can scaffold cognition, but score-bearing content must be sourced/reviewable/quarantined until validated. *Rests on:* Chen 2026 (+11.9pp transfer-explanation, MC ns, fewer problems); LearnLM/Eedi (+5.5pp, 76.4% minimal edits); SmartFlash editability; MyAnswerChecker/AnkiAIUtils; BMC "all faulty"; GPT-4 math −17% unassisted; parent AI-lane separation.
